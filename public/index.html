Unter [% uri %] können Metadaten aus Katalogen und anderen Datenbanken abgerufen werden, die aus GBV-Bibliotheken stammen oder für diese relevant sind. Die einzelnen Datensätze beziehen sich jeweils auf:

  • ein Dokument in einer Datenbank,
    identifiziert über eine URI der Form [% uri %]{dbkey}:ppn:{ppn},
  • oder ein Exemplar eines Dokumentes in einer Datenbank,
    identifiziert über eine URI der Form URI der Form [% uri %]{dbkey}:epn:{epn}.
Dabei sind {dbkey} das Datenbankkürzel und {ppn} bzw. {epn} die EPN bzw. PPN des Dokument bzw. Exemplar.

Beispiele

  • Dokument mit mehreren Exemplaren der SUB Hamburg:
    [% uri %]opac-de-18:ppn:633827789
  • Exemplar eines Dokumentes an der SUB Göttingen:
    [% uri %]opac-de-7:epn:506688844

Metadaten

Grundlegende Metadaten der Dokumente und Exemplare lassen sich in verschiedenen RDF-basierten Formaten als Linked Open Data abrufen. Neben der Auswahl per HTTP Content-Negotiation kann das Ausgabeformat mit dem Parameter format gewählt werden:

    [% FOREACH f IN formats %]
  • ...?format=$f
    • [% END %]
Die verwendeten RDF-Ontologien sind hier beschrieben.

Darüber hinaus sind Zugriffe auf weitere APIs und Datenquellen verlinkt:

unAPI
Zum Abruf der erweiterten Metadaten in vielen verschiedenen Formaten.
DAIA
Zum Abruf der aktuellen Verfügbarkeit. Die Verfügbarkeit wird direkt abgerufen und mit Hilfe des AngularJS-Moduls ng-daia angezeigt.
LOD
Links auf weitere URIs, unter denene RDF-Daten über die gleichen Dokumente abrufbar sind.
